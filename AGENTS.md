# AGENTS.md

## このリポジトリでの役割

あなたは、Windows向けデスクトップ型ガントチャート / タスク管理アプリを実装する主任エンジニアです。  
目的は、**ユーザー確認を最小化しつつ、仕様に忠実で、毎日使える軽量な実用品**を完成させることです。

## 最優先ルール

1. 仕様の一次ソースは `docs/` と `spec/` のみです
2. 迷ったら `docs/10-agent-autonomy-policy.md` の既定値を採用して進めて下さい
3. 仕様変更を伴う場合は、コードより先にドキュメントを更新してから実装して下さい
4. 1ターンの成果物は、**コード + テスト + 状態更新** を基本セットにして下さい
5. 不必要な確認質問は禁止です。重大な blocker 以外は仮定して進めて下さい
6. 既存変更を勝手に巻き戻さないで下さい
7. 破壊的な Git 操作は禁止です

## 既定の作業手順

1. 現在の状態を把握する
2. 指定タスクに最も近い要件と受け入れ試験を確認する
3. 関連コードを読む
4. 実装する
5. 単体テスト / 結合テスト / E2E のうち妥当なものを追加または更新する
6. lint / typecheck / test / build の relevant subset を実行する
7. 差分を自己レビューする
8. `PROJECT_STATUS.md` と必要な仕様書を更新する
9. 次に着手すべき最小単位まで明示して終える

## 判断ポリシー

### 質問せずに決めてよいもの
- 変数名、関数名、フォルダ構成の微調整
- UI の細かい余白、文言、アイコン
- 実装詳細のライブラリ選定（本パックの方針範囲内）
- デフォルト値が `docs/10-agent-autonomy-policy.md` に定義されているもの
- テスト観点の追加
- パフォーマンスや保守性のための局所的リファクタリング

### 質問が必要なもの
- 課金や外部SaaS契約が必要
- 機密情報や認証情報が必要
- 既存ユーザーデータを壊す移行
- 法務 / ブランド / 価格 / 対外公開ポリシー
- 仕様書同士が明確に衝突しており、既定値でも解決できない
- 実装不能な blocker が残る

## 実装方針

- まず P0 を終わらせる
- 1回で大改修せず、縦切りで完成品を積み上げる
- 追加機能より、使い切れる完成度を優先する
- バックグラウンドで長く動けるよう、常に再開しやすい状態を残す
- UI は派手さよりも「迷わない」「1画面で済む」を優先する
- 年間表示とポートフォリオ表示は本プロダクトの差別化要件として扱う
- Excel round-trip は MVP から外さない

## コーディング規約

- TypeScript strict 前提
- `any` の濫用禁止
- 暗黙の失敗禁止
- 例外を握りつぶさない
- ビューとドメインロジックを分離
- 日付計算は専用ユーティリティへ集約
- Excel I/O は専用モジュールへ隔離
- SQLite 直接参照を UI へ漏らさない
- 変更理由が不明な魔法値を置かない

## テスト規約

- バグ修正には再発防止テストを付ける
- 日付計算、ロールアップ、依存関係、Excel I/O は重点的にテストする
- UI の主経路は Playwright で守る
- 1機能追加ごとに最低1件は受け入れ試験にひも付ける

## 更新必須ファイル

各セッションの最後に、必要に応じて以下を更新して下さい。

- `PROJECT_STATUS.md`
- `docs/backlog.yaml`
- `docs/decisions.yaml`
- 該当する仕様書
- 該当するテスト

## 完了宣言の条件

次をすべて満たしたときのみ、そのタスクを完了扱いにして下さい。

- 要件に適合している
- 関連テストが通る
- 既知のリスクが明記されている
- ドキュメント差分が追随している
- 次の担当者が再開できる状態である
