# 02 Product Requirements

## 2.1 対象プラットフォーム
- Windows 11 を主対象
- Windows 10 は努力目標
- オフライン利用を第一とする
- 単一ユーザー利用を MVP の前提とする

## 2.2 ペルソナ

### ペルソナA: 複数案件を抱える実務担当
- 案件数: 5〜20
- 大きな案件はさらに 20〜200 タスクに分解される
- 日程変更が頻繁
- 共有は最後に Excel へ出す

### ペルソナB: ズボラ運用の個人マネージャ
- 1日に複数回更新はしない
- 今日やることと年間見通しを同じツールで見たい
- 設定画面が多いと使わなくなる

## 2.3 主要ジョブ

1. 思いついたタスクを雑に入れる
2. 大きなプロジェクトの内訳を作る / 畳む / 見直す
3. ドラッグで日程をずらす
4. 複数プロジェクトの危険箇所を見る
5. 年間計画 / FY 計画を眺める
6. Excel に出す / Excel から戻す

## 2.4 必須要件

### PR-001 Quick Capture
- タイトルだけで作成可能
- 日付・時間・所要時間・タグ・担当・プロジェクト名を同一行で解析
- 解析不能部分は失わずメモへ残す
- 入力後は自動保存

### PR-002 Inbox / Today
- 未計画タスクを Inbox に置ける
- 期限切れ、今日、今週、完了待ちを一目で見られる
- 期限切れ項目を一括で「今日」「明日」「今週末」に送れる

### PR-003 Project と WBS
- 1つのプロジェクトに階層タスクを持てる
- 階層は最低 6 レベル対応
- 展開 / 折りたたみができる
- 子の進捗から親をロールアップできる

### PR-004 Portfolio
- 複数の大規模プロジェクトを横断表示
- 各プロジェクトについて、進捗、期限超過件数、次マイルストーン、最終更新日時、リスク状態を一覧できる
- 行展開で主要フェーズまで見られる

### PR-005 Gantt and Rescheduling
- 日 / 週 / 月 / 四半期 / 年 / FY 表示
- バー移動、バー伸縮、マイルストーン移動
- 範囲変更時の自動保存
- 子タスク連動 / 後続連動のスコープ選択

### PR-006 Long-term Roadmap
- 月単位で見られる年間表示
- FY 開始月を設定可能
- 年間表示では詳細日付よりも大枠把握を優先
- マイルストーンは視認しやすい形で表示

### PR-007 Templates and Recurrence
- 繰り返しタスクに対応
- プロジェクト雛形 / WBS 雛形を保存できる
- 月初、週次、毎営業日などの定型生成に対応

### PR-008 Excel Round-trip
- XLSX を標準形式とする
- 出力は `Dashboard`, `Tasks`, `Gantt_View`, `MasterData`
- `Tasks` シートを canonical import/export とする
- 内部IDによる更新 import に対応する
- 新規行追加による新規タスク作成に対応する

### PR-009 Local First
- データはローカル保存
- 自動保存
- ローカルバックアップ
- クラッシュ後も最新に近い状態へ戻れる

### PR-010 Search and Filter
- タイトル、タグ、担当、プロジェクト、状態で検索可能
- 未計画 / 期限切れ / 進行中 / 完了を切り替え可能
- Portfolio と Year view でもフィルタ可能

## 2.5 非機能要件

### NFR-001 性能
- 10,000 アイテム規模でも起動 3 秒以内を目標
- 3,000 行の WBS 表示で実用的なスクロール性能
- 5,000 行の Excel export を 10 秒以内目標

### NFR-002 信頼性
- 自動保存間隔は 500ms 以内の debounce
- 重要操作は Undo/Redo 対応
- エラー時も入力内容を失わない

### NFR-003 保守性
- UI / domain / infra を分離
- 日付計算・ロールアップ・Excel I/O を独立モジュール化
- TypeScript strict を前提とする

### NFR-004 ユーザビリティ
- 主要機能は 1〜2 クリックで到達
- 保存ボタン依存を避ける
- ダイアログ乱立を避ける

## 2.6 既定の前提
- UI 言語は日本語優先
- タイムゾーンは OS ローカル
- 週の開始は月曜
- FY 開始月は初期値 4 月
- 稼働日は月〜金
- 祝日は後から追加できるが MVP では任意入力またはファイル取込対応まで
